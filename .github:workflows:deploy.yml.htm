Si el deploy desde Netlify funciona correctamente pero desde GitHub no carga el home, pueden estar pasando varias cosas. Aquí algunas posibles razones y soluciones:

1. Diferencias en la Configuración de Netlify

  • Netlify detecta automáticamente la configuración cuando haces deploy desde su propia plataforma, pero si lo haces desde GitHub, debes asegurarte de que los archivos de configuración sean correctos.
  • Verifica el archivo netlify.toml (si lo usas) y asegúrate de que la ruta de construcción (publish y build) es correcta.

2. Rutas y Configuración de React/Vite/Next.js (o cualquier framework)

  • Si usas React con React Router, necesitas un _redirects en public/ para manejar las rutas en Netlify:

    bash
    /*    /index.html   200
    • También puedes configurarlo en netlify.toml:
      toml
      [[redirects]]
from = "/*"
to = "/index.html"
status = 200

  • Si usas Next.js, verifica que next.config.js tenga:

    js
    module.exports = {
  trailingSlash: true, 
  output: 'export'
};

    (si es un sitio estático generado con next export)

3. Diferencias en las Rutas de Archivos

Si en Netlify funciona pero en GitHub Pages no, puede ser un problema de rutas. GitHub Pages sirve archivos desde /repo-name/ en lugar de /, lo que puede afectar las rutas de CSS, JS e imágenes.

  • Para React/Vite, agrega en vite.config.js:
    js
    export default defineConfig({
  base: "/nombre-del-repositorio/"
});
  • Para Next.js, en next.config.js:
    js
    basePath: "/nombre-del-repositorio"

4. Archivos Ignorados en GitHub

Asegúrate de que public/ y dist/ (o out/ en Next.js) no estén en .gitignore, ya que GitHub necesita estos archivos para servir el sitio.

5. Errores en la Consola

Revisa la consola del navegador (F12 → Console) para ver si hay errores de archivos no encontrados (404) o CORS.

GitHub no muestra la carpeta dist/ a menos que la agregues y subas manualmente, lo cual no es necesario en la mayoría de los casos. Aquí te explico cómo hacer el deploy correctamente en GitHub Pages con Vite:

1. Usa vite-plugin-static para Deploy Automático

En lugar de preocuparte por subir dist/ manualmente, puedes usar vite-plugin-static para que Vite genere y publique automáticamente en GitHub Pages.

Pasos:

1️⃣ Instala el plugin

sh
npm install -D vite-plugin-static

2️⃣ Modifica vite.config.ts
Añade este código:

ts
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import { staticPlugin } from 'vite-plugin-static';

export default defineConfig({
  base: "/workinfi_resume/", // Nombre del repo en GitHub
  plugins: [react(), staticPlugin()],
});

3️⃣ Corre el build y publica en GitHub Pages

sh
npm run build
git add dist -f
git commit -m "Deploy to GitHub Pages"
git subtree push --prefix dist origin gh-pages

Esto enviará el contenido de dist/ a la rama gh-pages, que es la que usa GitHub Pages.

2. Alternativa: Configurar GitHub Actions

Si no quieres hacer esto manualmente cada vez, puedes usar GitHub Actions para que el deploy sea automático.

1️⃣ Crea un archivo en .github/workflows/deploy.yml con este contenido:

yaml
name: Deploy to GitHub Pages

on:
  push:
    branches:
      - main  # Cambia "main" si usas otra rama principal

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v3

      - name: Install and Build
        run: |
          npm install
          npm run build

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist

2️⃣ Guarda los cambios, súbelos a GitHub y cada vez que hagas push a main, el sitio se actualizará automáticamente.

Cómo Verificar si el Deploy Funciona

  • Ve a Settings → Pages en tu repositorio.
  • Asegúrate de que el deploy está en la rama gh-pages.
  • La URL de tu sitio debería ser:
    arduino
    https://yourusername.github.io/workinfi_resume/

Prueba esto y dime si sigues teniendo problemas. 🚀
